Modal Gallery

Modal Gallery is the core component of @ks89/angular-modal-gallery. It displays a navigable collection of Images with action buttons, previews, and many other customizable features.


Live demo



API

Follow the Getting Started guide to install @ks89/angular-modal-gallery and all required dependencies.

To use the modal gallery in your application, invoke the .open(...) method of ModalGalleryService. This method can be called from anywhere in your component, for example in response to a button click.

Unlike earlier versions, which required adding a selector to the HTML template, the modal gallery is now an Angular service. It can be opened programmatically without adding any UI element to your template, or triggered automatically in response to any event.

Inputs

id
Type: number
A mandatory unique id >= 0 of the current modal gallery, used by GalleryService.
images
Type: Image[]
The mandatory array of Image objects, that represent the modal images. In fact, an Image is an object with image source paths and other additional configuration parameters. For more info check Image. Avoid using this.images.push(img) to modify this array. Instead, re-assign it to trigger change detection. For example: this.images = [...this.images, newImage] creates a new array with the additional image appended.
Warning: if you need to add the same image (same URL) multiple times in the same modal gallery, you must append a random but unique query parameter to the URL to prevent caching issues. For more info, please read this issue.
currentImage
Type: Image
The image to display when the modal gallery opens. Pass one of the elements from the images array. For example, to open the first image, pass images[0].
config
Type: ModalLibConfigoptional

Interface to configure modal gallery.

Outputs

close

An ImageModalEvent right before that the modal view closes.

It contains an Action enum and a result (boolean or number)

  • action: Action that can be NORMAL, KEYBOARD or CLICK
  • result: true
show

An ImageModalEvent right before that the modal image will be visible.

It contains an Action enum and a result (boolean or number).

Every time the current image changes, a new ImageModalEvent will be emitted.

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD, Action.SWIPE, Action.AUTOPLAY or Action.SWIPE
  • result: number (index of an image. Not the index of the array, but incremented by 1)
firstImage

An ImageModalEvent right before that the FIRST image in modalImages is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD, Action.SWIPE, Action.AUTOPLAY or Action.SWIPE
  • result: true
lastImage

An ImageModalEvent right before that the LAST image in modalImages is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD, Action.SWIPE, Action.AUTOPLAY or Action.SWIPE
  • result: true
hasData

An ImageModalEvent right after the initialization of modalImages.

It contains an Action enum and a result (boolean or number)

This event is useful to know when @ks89/angular-modal-gallery is ready.

  • action: Action.LOAD
  • result: true
buttonBeforeHook

A ButtonEvent right before the action related to an upper button.

It contains an Action enum, the current Image and the ButtonConfig that triggered this event

This is a before-hook event, allowing you to execute custom logic prior to any button-triggered action. The full ButtonConfig payload enables branching behavior based on the button type.

  • action: Action.LOAD
  • button: ButtonConfig
  • image: Image
buttonAfterHook

A ButtonEvent right after the action related to an upper button.

It contains an Action enum, the current Image and the ButtonConfig that triggered this event

This is an after-hook event, allowing you to execute custom logic following any button-triggered action. The full ButtonConfig payload enables branching behavior based on the button type.

  • action: Action.LOAD
  • button: ButtonConfig
  • image: Image


ModalImage

Interface that represent the modal image.


img
Type: string | SafeResourceUrl

The mandatory source path of the modal image. For base64 image support, refer to the official example.

If multiple non-base64 images share the same URL, append a unique query parameter (e.g., ?0, ?1) to prevent caching issues. For more information, refer to the official "Same Images" example.
description
Type: stringoptional

Optional description text. Note: this is not the full description, but only the trailing portion following the image count. For example: Image 2/12 - description. To omit this feature, simply exclude this property.
extUrl
Type: stringoptional

Optional external URL. Must be a valid URL, as @ks89/angular-modal-gallery navigates directly to it. To omit this feature, exclude this property.

Type: stringoptional

Optional title text. It sets 'title' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
alt
Type: stringoptional

Optional alt text. It sets 'alt' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
arialLabel
Type: stringoptional

Optional aria label text. It sets 'aria-label' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
invertSwipe
Type: booleanoptional

Optional boolean property to invert the default swipe direction on devices with touchscreen.
sources

Optional array of sources for defining srcset values at responsive media breakpoints.

CurrentImageConfig

Interface to customize current image in modal gallery.


downloadable
Type: booleanoptional

When true, the current modal image can always be downloaded via the keyboard shortcut (ctrl+s or cmd+s). The download button is only visible when showDownloadButton=true. When false, all download functionality is disabled. Defaults to false.

Note: keyboard shortcuts are unavailable on smartphones and tablets. To enable image downloads on touch devices, set downloadable=true and use ButtonsStrategy.SIMPLE or higher to display the download button.
loadingConfig
Type: LoadingConfigoptional

Interface to hide/show and to configure the modal loading spinner. { enable: true, type: 1 } by default.
description
Type: Description interfaceoptional

Interface for specifying a custom description, either as a complete string or as an object with individual string properties. The full string form accepts a complete description (e.g., 'Image 1/2 - My family'). The object form supports imageText (default: 'Image'), numberSeparator (default: '/'), and beforeTextDescription (default: ' - '). HTML content is supported in descriptions; however, Angular sanitizes potentially unsafe markup, so only safe HTML elements are permitted.

To customize individual parts of the description, refer to this pattern: imageText 2 numberSeparator 12 beforeTextDescription description text. The three configurable properties correspond to the prefix, separator, and preceding text. 2 is the index of the currently displayed image, 12 is the total image count, and the description text comes from the Image input object.

Note: values are concatenated without additional spacing. To produce output such as 'text - text', specify ' - ' (including the surrounding spaces).

From version 5.4.0 onwards, this interface can also be used to customize the description's visual style.
navigateOnClick
Type: booleanoptional

If false, click events on the current image will be ignored, otherwise will navigate to the next image.
navigateOnClick
Type: booleanoptional

If false, click events on the current image will be ignored, otherwise will navigate to the next image.

ModalGalleryService

Service to interact with modal gallery.


open( config )
Input types: config: ModalGalleryConfig
Return types: ModalGalleryRef | undefined to listen for events.

Opens the modal gallery for the specified gallery id at the given image index. Both the index and the gallery ID must be >= 0. Gallery IDs must be unique across the entire page; each instance of the modal gallery requires a distinct ID.
close( id, clickOutside )
Input types: id: number | undefined and clickOutside: boolean

Closes the modal gallery identified by the specified id. Pass false for clickOutside in most cases. Pass true only when the close should be conditional on enableCloseOutside being set to true in ModalLibConfig.
updateModalImages ( images )
Input types: images: Images[]

Method to update the current images array with another one passed as input.

buttonBeforeHook$

buttonAfterHook$

LoadingType

Enum representing the loading spinner type. Possible values:


  • STANDARD: the default spinner
  • CIRCULAR: a circular spinner
  • BARS: spinner composed of animated vertical bars
  • DOTS: spinner composed of animated dots
  • CUBE_FLIPPING: a flipping cube spinner
  • CIRCLES: spinner composed of multiple circles
  • EXPLOSING_SQUARES: spinner composed of 4 outward-expanding squares

KeyboardEnum

Enum representing keyboard key codes. Possible values:


  • ESC: 'Escape'
  • LEFT_ARROW: 'ArrowLeft'
  • RIGHT_ARROW: 'ArrowRight'
  • UP_ARROW: 'ArrowUp'
  • DOWN_ARROW: 'ArrowDown'
