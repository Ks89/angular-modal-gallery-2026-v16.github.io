Description interfaces

Description interfaces shared across all components of this library.

Description

Interface to set a config image description.


strategy
Type: DescriptionStrategy

Enum to choose when description should be visible. ALWAYS_VISIBLE by default.
imageText
Type: stringoptional

The prefix of the description text. Defaults to 'Image'. It can be localized by specifying a custom value, such as 'Immagine' (Italian), 'Bild' (German), 'Imagen' (Spanish), or an empty string ''.

Example: `imageText 2/12 - bla bla`
numberSeparator
Type: stringoptional

It's the text between the current image number and the total number of images.

Example: `Image 2numberSeparator12 - bla bla`
beforeTextDescription
Type: stringoptional

It's the text between the total number of images and the final description text

Example: `Image 2/12beforeTextDescriptionbla bla`
customFullDescription
Type: stringoptional

The complete description string. When set, all other description properties are ignored.

For a dynamic description that incorporates the current image index or total count, construct it manually. Otherwise, use the other properties of this interface to customize the default description.
style
Type: DescriptionStyleoptional

Interface to change background color, text color, margins, positions, sizes and so on.

All properties are optional and some of them have default values. Check DescriptionStyle for more information.

DescriptionStrategy

Enum controlling the visibility of the description. Possible values:


  • ALWAYS_HIDDEN: description is always hidden
  • ALWAYS_VISIBLE: description is always visible
  • HIDE_IF_EMPTY: description is visible only when a non-empty value is provided
