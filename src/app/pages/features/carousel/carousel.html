Carousel

The carousel component builds responsive, auto-advancing image galleries. Like other library components, it accepts a collection of Images and various configuration objects. It renders a large image (full or constrained width) with left and right navigation arrows, cycling through images at a configurable interval. Previews can optionally be displayed above or below the main image. Additional configurable features include ARIA support, keyboard navigation, and custom sizes. A notable feature is the integration with the modal gallery: the carousel can be configured to open the current image using ModalGalleryService.

Inputs

id
Type: number
A mandatory unique id >= 0 of the current carousel. Although CarouselComponent does not use this value, it must be unique because a carousel can open a modal image using ModalGalleryService.
images
Type: Image[]

The mandatory array of Image objects, that represent both the thumb gallery and the modal images. In fact, an Image is an object with both thumb and image source paths with other additional configuration parameters. For more info check Image.

The carousel uses the image path assigned to the modal property of the Image class. This is by design, for historical reasons and to ensure better integration with the modal gallery.

Avoid using this.images.push(img) to modify this array. Instead, re-assign it to trigger change detection. For example: this.images = [...this.images, newImage] creates a new array with the additional image appended.
config
Type: CarouselLibConfigoptional

Interface to configure carousel gallery.

Outputs

changeImage

An ImageEvent right after the current image is changed.

It contains an Action enum and a result (boolean or number).

Every time the current image changes, a new ImageEvent will be emitted.

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: number (index of an image. Not the index of the array, but incremented by 1)
firstImage

An ImageEvent right after that the FIRST image in images is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: true
lastImage

An ImageEvent right after that the LAST image in images is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: true
clickImage

A number that represents the index of the clicked image to open modal-gallery (modalGalleryEnable in CarouselConfig must be true).


CarouselConfig

Interface to configure the carousel, like to change sizes, hide some features and so on.


maxWidth
Type: string

Maximum width of the carousel. It's used to set the CSS property maxWidth. '100%' (full width) by default.
maxHeight
Type: string

Maximum height of the carousel. It's used to set the CSS property maxHeight. '400px' by default.
showArrows
Type: boolean

When false, both left and right navigation arrows are hidden. Defaults to true.
objectFit
Type: string

Sets the CSS object-fit property. For more information, refer to the MDN documentation. Defaults to 'cover'.
keyboardEnable
Type: boolean

When false, keyboard navigation is disabled. Defaults to true.
modalGalleryEnable
Type: boolean

When true, clicking the current image opens the modal gallery. Defaults to false.

PlayConfig

Interface for configuring auto-navigation behavior, including the autoPlay setting and transition interval in milliseconds.


autoPlay
Type: boolean

When false, autoPlay is disabled. Defaults to true.
interval
Type: number

A non-negative integer specifying the interval in milliseconds between image transitions. Applicable only when autoPlay is enabled. Defaults to 5000.
pauseOnHover
Type: boolean

If false, the carousel will not pause when the mouse pointer is moved over it. true by default.

CarouselImageConfig

Interface for configuring the carousel's current image, including the description and swipe direction.


description
Type: Descriptionoptional

Optional description text. For more info, please check Description interface.
invertSwipe
Type: booleanoptional

Optional boolean property to invert the default swipe direction on devices with touchscreen.

ImageEvent

The payload of events related to action, emitted by this library.


galleryId
Type: number

Unique id of the current carousel.
action
Type: Action

It's the enum Action
result
Type: number | boolean

The result or data payload of the emitted event. For example, it may represent the index of the current image or a boolean indicating whether images are ready to display.
